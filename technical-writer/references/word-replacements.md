# Word replacements

This list grows over time. Add entries when you identify a vague or ambiguous
term and its preferred replacement. Always apply these substitutions during
Pass 3 (Refine) and Pass 4 Words level (Review).

## Format

Each entry: `avoid | use | reason`

## Current replacements

| Avoid | Use | Reason |
|-------|-----|--------|
| allow | enable | "Allow" implies permission; "enable" implies capability. |
| may | can or might | "May" is ambiguous between permission and possibility. Use "can" for capability, "might" for uncertainty. |
| e.g. | for example | Not all readers know Latin abbreviations. |
| i.e. | that is | Not all readers know Latin abbreviations. |
| etc. | (omit or list explicitly) | Vague; list the actual items or restructure. |
| above | (use a heading reference or "the previous section") | "Above" breaks in non-linear reading. |
| below | (use a heading reference or "the following section") | "Below" breaks in non-linear reading. |
| simply | (omit) | Implies the task is easy; condescending to readers who find it hard. |
| just | (omit) | Same as "simply". |
| basically | (omit) | Filler with no meaning. |
| obviously | (omit) | Condescending. |
| easy | (omit or restructure) | Condescending if the reader finds it hard. |
| please | (omit) | Overly formal or conversational depending on context. |
| reach out | contact | Idiomatic. |
| kick off | start | Idiomatic. |
| spin up | start or launch | Jargon. |
| tear down | remove or destroy | Jargon in infrastructure contexts. |
| leverage | use | Overused business jargon. |
| utilise | use | Unnecessarily complex. |
| in order to | to | Wordy. |
| is able to | can | Wordy. |
| due to the fact that | because | Wordy. |
| at this point in time | now | Wordy. |
| master (branch/node) | primary | Non-inclusive legacy term. |
| slave (node/device) | secondary | Non-inclusive legacy term. |
| whitelist | allowlist | Non-inclusive legacy term. |
| blacklist | denylist | Non-inclusive legacy term. |
| sanity check | validation or quick check | Non-inclusive. |
| dummy | placeholder or example | Can be non-inclusive depending on context. |

## Adding entries

When you identify a new replacement during a review pass, add it here with
the `avoid`, `use`, and `reason` columns filled in. Keep the table sorted
alphabetically by the "avoid" column.
